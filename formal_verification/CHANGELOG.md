# Changelog - Formal Verification Manifesto

All notable changes to the Formal Verification Manifesto will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1] - 2024

### Added
- Tool selection decision tree
- Cost-benefit analysis framework
- Learning path recommendations (30-day, 3-month, 6-month)
- Implementation guide with phased approach

### Changed
- Expanded tooling section to include emerging proof assistants
- Updated examples to reflect modern formal verification practices
- Enhanced Curry-Howard correspondence explanations

### Fixed
- Clarified termination checking requirements
- Corrected mathematical notation in refinement types section

## [1.0] - 2024

### Added
- Initial release of Formal Verification Manifesto
- 16 foundational principles covering:
  - Specification as Foundation
  - Types as Propositions (Curry-Howard)
  - Totality & Termination
  - Refinement Types
  - Separation Logic
  - Proof Automation
  - Verified Compilation
  - Incremental Formalization
- Tool ecosystem coverage (Coq, Isabelle, Lean, Agda, F*, Dafny, TLA+, SPARK Ada)
- Best fit guidance for safety-critical and security-critical systems
- Integration with other manifestos (Vibe Coding, Security Hardening)
